# 気候指数 WebGIS – 暖かさの指数・年最深積雪

中部・東海・北陸・関東・東北地方の平年値気温データから暖かさの指数（温量指数, WI）と
年最深積雪を算出し、Leafletを用いてWebGISとして可視化するプロジェクト。

---

## 概要

### 暖かさの指数（Warmth Index, WI）

吉良龍夫が提唱した気候指数で、植生帯の区分に広く用いられる。

$$WI = \sum_{i=1}^{12} \max(T_i - t, \; 0)$$

- $T_i$: 月平均気温（°C）
- $t$: しきい値（本実装では **5°C**）

#### 植生帯の目安（吉良の気候帯区分）

| WI 範囲 | 植生帯 |
|---------|--------|
| WI < 15 | 高山帯 |
| 15 〜 45 | 亜高山帯 |
| 45 〜 85 | 冷温帯（山地帯） |
| 85 〜 130 | 暖温帯上部 |
| 130 〜 180 | 暖温帯下部 |
| WI ≥ 180 | 亜熱帯 |

### 年最深積雪

平年値（1991–2020年）の年最大積雪深（cm）。
積雪 0 cm の地点および欠損値は地図上に表示しない。

### 使用フィールド（G02-22フォーマット）

G02-22では月平均気温は以下のフィールドに格納されている（単位: 0.1°C）。

| 月 | フィールド | 月 | フィールド |
|----|-----------|----|-----------| 
| 1月 | G02_017 | 7月 | G02_035 |
| 2月 | G02_020 | 8月 | G02_038 |
| 3月 | G02_023 | 9月 | G02_041 |
| 4月 | G02_026 | 10月 | G02_044 |
| 5月 | G02_029 | 11月 | G02_047 |
| 6月 | G02_032 | 12月 | G02_050 |

| フィールド | 内容 | 単位 |
|-----------|------|------|
| G02_058 | 年最深積雪 | cm |

> **注意**: G02-12（旧版）では月平均気温が G02_002〜G02_013 に格納されていたが、
> G02-22（新版）ではフィールド構成が変更され、月降水量が先頭フィールドになった。
> 本実装は G02-22 フォーマットに対応している。

### 中間ファイルのフォーマット

**mon_temp.dat**（スペース区切り）:
```
meshcode x y t1 t2 t3 t4 t5 t6 t7 t8 t9 t10 t11 t12
52370008 137.00625 34.67083 5.9 6.2 9.4 ... 8.1
```
- `meshcode`: 8桁の3次メッシュコード
- `x`: 経度（°E, JGD2000/WGS84）
- `y`: 緯度（°N, JGD2000/WGS84）
- `t1〜t12`: 1〜12月の月平均気温（°C）

**wi.dat**（スペース区切り）:
```
meshcode x y wi
52370008 137.00625 34.67083 123.4
```
- `wi`: 暖かさの指数（°C·月）, しきい値=5°C
- メッシュ数: 170,959
- WI 範囲: 3.3〜149.8 °C·月

**snow.dat**（スペース区切り）:
```
meshcode x y snow
52360709 136.99375 34.67083 1
```
- `snow`: 年最深積雪（cm）
- メッシュ数: 166,066（有効データのあるメッシュ）
- 積雪範囲: 0〜531 cm

## 植生 WebGIS（meshveg.html）

wi.html を基に、環境省第3次メッシュ植生データからオオシラビソ（アオモリトドマツ）の
分布を追加したWebGIS。

### 植生データ処理フロー

```
veg/veg05mesh.csv      (メッシュコード, 群落コード)
veg/veg_gunraku.csv    (群落コード, 群落名, 集約群落コード)
         ↓ process_veg.py
veg/aomori_todomatsu.json  (lat, lon, 群落コード, 群落名)
         ↓ create_meshveg.py
meshveg.html
```

### オオシラビソ関連群落

wi.dat の対象範囲（緯度 34.67°〜41.99°, 経度 136.01°〜141.99°）内で抽出された
1,636 メッシュのデータを使用。

| 群落コード | 群落名 | 色 |
|-----------|--------|-----|
| 20501A | オオシラビソ群集 | 濃緑 (#006400) |
| 20501B | アオモリトドマツ群落 | 緑 (#228B22) |
| 20502A | シラビソ−オオシラビソ群集 | 中緑 (#3CB371) |
| 21400 | オオシラビソ−ダケカンバ林 | 淡緑 (#8FBC8F) |

### 緯度経度の分布範囲

- 緯度: 35.30°〜40.69°（本州中部〜北東北）
- 経度: 136.69°〜141.54°

### Web地図（meshveg.html）の機能

| 機能 | 説明 |
|------|------|
| オオシラビソ表示 | 群落コードごとに緑系4色で3次メッシュセルを描画（レイヤーコントロールで有効化） |
| WI 色分け表示 | WI値に応じて6段階の色で描画（デフォルト表示） |
| 積雪色分け表示 | 年最深積雪を5段階の青系色で描画（レイヤーコントロールで有効化） |
| クリック | 表示中の全レイヤーの値（WI・積雪・群落名）をポップアップ表示 |
| 植生凡例 | 画面右下に群落コードと色の対応を表示 |
| WI 凡例 | 画面左下に植生帯区分の凡例を表示 |

### 実行手順（植生 WebGIS）

```bash
# 前提: process_veg.py でオオシラビソメッシュデータを生成
python process_veg.py   # → veg/aomori_todomatsu.json を生成

# meshveg.html を生成
python create_meshveg.py
```

## 実行手順

### 前提条件

- Python 3.x（追加ライブラリ不要、標準ライブラリのみ使用）
- インターネット接続（地図タイル取得のため）

### 手順

```bash
# プロジェクトフォルダへ移動
cd wi_project

# ステップ1: 対象メッシュのZIPをダウンロード
python3 download.py

# ステップ2: 月平均気温・年最深積雪データの作成と暖かさの指数の算出
#   → mon_temp.dat, snow.dat, wi.dat を生成
python3 process.py

# ステップ3: Leaflet Web地図の生成
python3 create_html.py

# Web地図を開く（例: Windows）
start wi.html
```

## 補足: 座標変換

3次メッシュコード (8桁) から中心座標を算出する計算式（`process.py` に実装）:

```
code = "PPQQRSTU"  (8桁)
lat_sw  = PP / 1.5 + R / 12 + T / 120
lon_sw  = (QQ + 100) + S / 8 + U / 80
lat_ctr = lat_sw + 1/240  (セル高さの半分)
lon_ctr = lon_sw + 1/160  (セル幅の半分)
```
